# 每日复习策略说明

更新时间：2026-06-24

本文档总结当前 ExamDeck「每日复习」的实际策略，依据源码中的 `DailyReviewSession`、`buildDailyReviewPlan`、`getReviewIntervalDays`、`startDailyReview` 等逻辑整理。

## 一句话结论

当前每日复习支持「当前题库复习」和「全题库每日复习」。首页的每日复习入口默认使用全题库模式：把所有普通题库合并成一个去重后的复习范围，已学习、未斩题、到期的题目会进入每日复习候选队列。

## 复习对象范围

当前题库复习会从当前选中的题库中取题：

- 当前题库：`activeDeck`
- 当前题库题目：`activeQuestions`
- 已斩题会被排除：`activeReviewQuestions = activeQuestions.filter(question => !slashedQuestionSet.has(question.id))`

所以：

- 在「技师题」里点每日复习，只复习技师题。
- 在「高级工」里点每日复习，只复习高级工。
- 已斩题默认视为已熟练，不再进入每日复习。

全题库每日复习会额外构造一个虚拟题库：

- 虚拟题库名：全题库每日复习
- 虚拟题库 ID：deck_all_daily_review
- 合并范围：所有普通题库
- 排除范围：重难题这类派生题库，避免重复计入
- 合并方式：按题目 ID 去重后再生成每日复习队列

## 什么题算“已学习”

每日复习只处理已经做过的题。判断条件是：

```ts
stat && stat.seen > 0
```

也就是说，只要某题有答题记录，就算进入复习系统；完全没做过的题不会出现在每日复习中。

## 日期边界：凌晨 4 点

现在每日复习按凌晨 4 点作为一天的切换点，不按自然日 0 点切换。

实现逻辑是：

- 先把当前时间减去 4 小时。
- 再取本地日期。
- 对应的复习日从当天 4:00 开始。

这样做的效果是：

- 00:00 - 03:59 仍然算前一天。
- 到 04:00 后才进入新的复习日。
- 如果用户熬夜做题，不会在 0 点突然刷新队列。

## 到期时间计算

每道题的复习到期时间由最后一次答题时间和复习间隔决定：

```ts
dueDate = startOfReviewDay(lastSeenAt) + intervalDays
```

注意这里的 `startOfReviewDay` 是凌晨 4 点边界，不是 0 点。

举例：

- 6 月 23 日 23:00 做过一道题，间隔 1 天。
- 它的复习日开始时间是 6 月 23 日 04:00。
- 到期时间是 6 月 24 日 04:00。
- 所以 6 月 24 日 03:00 不会提前出现，6 月 24 日 04:00 后才进入复习。

## 复习间隔算法

复习间隔主要由 `correctStreak` 连续答对次数决定，再根据正确率和错误情况修正。

基础间隔表：

| 连对状态 | 基础间隔 |
|---:|---:|
| seen <= 1 | 1 天 |
| correctStreak = 0 | 1 天 |
| correctStreak = 1 | 2 天 |
| correctStreak = 2 | 4 天 |
| correctStreak = 3 | 7 天 |
| correctStreak = 4 | 15 天 |
| correctStreak = 5 | 30 天 |
| correctStreak = 6 | 60 天 |
| correctStreak = 7 | 120 天 |
| correctStreak = 8 | 240 天 |
| correctStreak >= 9 | 365 天 |

修正规则：

1. 正确率低于 60%：间隔减半，最少 1 天。
2. 正确率不低于 90% 且连续答对至少 3 次：间隔放大 1.25 倍。
3. 错误次数不少于正确次数：间隔最多 3 天。
4. 最近错过且连续答对不超过 1 次：间隔最多 2 天。
5. 最终间隔最少为 1 天。

## 每日复习队列生成

复习队列会筛选复习范围中：

1. 已学习的题。
2. 未被斩掉的题。
3. 到期时间小于等于当前复习日开始时间的题。

每个复习项包含：

- `questionId`
- `optionOrder`
- `selectedKeys`
- `dueAt`
- `intervalDays`
- `overdueDays`

其中 `overdueDays` 表示逾期天数，用于排序和统计。

## 队列排序规则与每日上限

每日复习队列按以下优先级排序：

1. 逾期天数多的排前面。
2. 错误次数多的排前面。
3. 正确率低的排前面。
4. 连续答对次数少的排前面。
5. 最近错过的题排前面。
6. 最早见过的题排前面。

每天最多生成 1000 道每日复习题：

- 如果到期题不超过 1000 道：全部进入今日队列。
- 如果到期题超过 1000 道：先按上面的优先级排序，再截取最需要复习的 1000 道。
- 页面统计会同时保留「实际今日队列」和「总到期数」的信息，方便知道是否被上限截断。

这意味着系统倾向于优先复习：

- 拖得久的题。
- 错得多的题。
- 还不稳定的题。
- 最近又错过的题。
- 很久之前学过的题。

## 答题记录如何写入

每日复习中，题目一旦被判定，就会立即写入学习记录：

- 判断题、单选题：选择后自动判定。
- 多选题：需要点击确认答案后判定。
- 判定后会更新：
  - `seen`
  - `correct`
  - `wrong`
  - `correctStreak`
  - `lastSeenAt`
  - `lastWrongAt`
  - 当日刷题统计 `dailyStats`

所以即使用户没有点「完成复习」，已经判定过的题也已经计入学习记录。

## 跨凌晨 4 点未完成的处理

现在新增的处理逻辑是：

如果用户打开着昨天的每日复习队列，并且时间跨过凌晨 4 点：

1. 系统会自动关闭旧复习 session。
2. 已经判定过的题保留答题记录。
3. 未判定的题不算完成。
4. 页面重新统计新一天的复习内容。
5. 用户再进入每日复习时，会看到新的到期队列。

这解决了“昨天未做完的复习内容今天仍然不刷新”的问题。

## 页面统计含义

每日复习页的统计来自 `buildDailyReviewSummary`：

- 已学习：当前题库中 `seen > 0` 的题数。
- 应复习：今日实际队列题数，最多 1000 道。
- 总到期：当前已经到期的候选题数，可能超过 1000。
- 逾期：今日实际队列中到期时间早于当前复习日的题数。
- 总逾期：全部候选中到期时间早于当前复习日的题数。
- 今日到期：应复习 - 逾期。

## 预测图逻辑

统计页里的复习预测同样使用复习到期时间：

- 以当前复习日 4:00 为第 0 天。
- 未来 31 天内每一天统计将到期的题数。
- 逾期题会归入今天。

## 当前策略的优点

- 简单、可解释，用户容易理解。
- 错题和不稳定题会被优先拉回来。
- 全题库每日复习不用在多个题库之间来回切换。
- 每日 1000 道上限能避免极端积压时生成不可控的复习量。
- 连续答对越多，间隔越长。
- 凌晨 4 点边界适合熬夜刷题场景。
- 已判定题即时入账，不依赖最终提交。

## 当前策略的限制

1. 没有按题型平衡复习；排序主要看逾期、错误、正确率、连对、错题时间和最后复习时间。
2. 多选题如果只选择但没确认，跨天时不会计入完成。
3. 已斩题不会进入每日复习，这适合“彻底掌握”的语义，但如果误斩需要手动取消。
4. 复习间隔是自定义规则，不是严格的 SM-2 / FSRS 算法。
5. 超过 1000 道的到期题会顺延到后续日期，今天只复习最需要的 1000 道。

## 可选优化方向

后续如果要继续优化，可以考虑：

1. 增加题型配比，避免全是同一题型。
2. 引入更成熟的间隔重复算法，例如 SM-2 或 FSRS。
3. 对跨天未完成 session 增加一条历史记录，方便回看“昨日完成了多少”。
4. 对多选题的“已选但未确认”增加提醒，防止用户误以为已经完成。
